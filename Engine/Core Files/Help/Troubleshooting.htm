Troubleshooting

A note from the author:

I’ve been working with EnCase and EnScript since 2003.  In all that time, I have come up with only one strong and steady rule: expect trouble.

Extracting data from evidence is tricky and troublesome.  There are an infinite number of possibilities and configurations in digital forensics, and it is simply not possible to account for all of them.  A fix for one problem may create a dozen new ones.

If you ever run into trouble, ask for help.  Chances are you’ve caught something that other people are frustrated by, and you may end up helping a lot of other people in the process.  Every bug report, error report, question about accuracy, etc helps make the C4P EnScript stronger in the long run.

Submitting an Error Report

When submitting an error report, please try to include the following:

·         The version of EnCase you are using

·         The version of the C4P EnScript you are using (compile date for older versions)

·         If you are asking about Pre-Categorization:

o    The database platform you are using

·         The version of Windows you are running on – especially 32-bit versus 64-bit

Answering these questions at the beginning will help eliminate a number of back and forth emails about the basics.
